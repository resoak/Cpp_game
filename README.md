# Logic Gate Defense

`Logic Gate Defense` 是一款以邏輯閘與神經網路為主題的 C++ 塔防遊戲。玩家需要在地圖上放置感測器、邏輯閘、感知器與砲塔，將訊號連成防禦網路，阻止病毒敵人突破並攻擊 CPU 核心。

遊戲使用 [raylib](https://www.raylib.com/) 繪製畫面與處理輸入，透過 CMake 建置。若系統沒有安裝 raylib，`CMakeLists.txt` 會使用 `FetchContent` 自動下載 raylib 5.0。

## 遊戲特色

- 邏輯閘塔防：用 `SENSOR -> AND/OR/XOR/NAND/PERCEPTRON -> CANNON` 建立可運作的訊號鏈。
- 感知器學習：感知器會在波次結束後根據戰鬥結果調整 `w1`、`w2` 與 `bias`。
- AI 敵情系統：敵人會統計兵種與路線存活率，後續波次會逐步適應玩家防線。
- 防禦建議 NN：右側資訊欄會顯示神經網路對防禦補強方向的建議。
- 威脅熱圖：擊殺位置會累積成熱圖，協助判斷火力集中區與薄弱區。
- 動態路線：每 3 波會輪換路線，Wave 7 起開放四方向入口，Wave 10 / 16 / 30 逐步擴展為多線進攻。
- 波次事件：包含蜂群、裝甲洪流、霧戰、精英突擊、護衛 Boss、再生軍、神速衝鋒、通訊中斷、變異體與圍城艦。
- 主動技能：每種非 CPU 元件都有可觸發技能，例如 EMP、超頻連射、全域標記與破甲反轉。
- 可選式教學關卡：主選單可進入教學章節選單，逐步練習基礎防線、各元件用途與多線防守。

## 系統需求

- CMake 3.16 以上
- 支援 C++17 的編譯器
  - Windows：MSVC、MinGW 或 Clang
  - Linux：GCC 或 Clang
  - macOS：Apple Clang
- Git 與網路連線：首次建置且本機沒有 raylib 時，CMake 需要下載 raylib

## 建置與執行

### Windows / Linux / macOS

```bash
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build --config Release
```

執行檔名稱為 `lgd`。

Linux 或 macOS：

```bash
./build/lgd
```

Windows 使用 MSVC 產生器時：

```powershell
.\build\Release\lgd.exe
```

Windows 使用單組態產生器，例如 MinGW 時：

```powershell
.\build\lgd.exe
```

## 操作方式

| 滑鼠點擊左側元件 | 選擇要放置的元件，再點擊地圖空格放置 |
| 滑鼠點擊已有元件 | 選取元件並查看資訊 |
| `C` | 選取元件後進入連線模式，再點擊目標元件建立或取消連線 |
| `U` | 升級選取元件 |
| `Q` | 發動選取元件的主動技能 |
| `Delete` | 移除選取元件，退還 50% 基礎費用 |
| `A` | 切換 AI 顧問提示 |

TRAINING 階段可按 `1`、`2`、`3`，或點擊左側選項，選擇一項波次結束獎勵。若倒數結束前未選擇，遊戲會自動套用第 1 項。

## 教學關卡

主選單可按 `T` 或點擊「教學關卡」進入章節選單。教學章節不鎖定進度，所有章節都可直接選擇；完成狀態只會在本次執行期間顯示為 `已完成` 或 `未完成`。

教學章節選單支援：

- 滑鼠點擊章節卡片開始教學。
- `1` 到 `9` 或數字鍵盤 `1` 到 `9` 直接選擇章節。
- 方向鍵移動目前選取章節。
- `Enter` / `Space` 開始目前選取章節。
- `Esc` 返回主選單。

目前教學章節：

| 章節 | 主題 | 對應敵人 | 重點 |
| --- | --- | --- | --- |
| 基礎防線 | SENSOR + CANNON | BASIC | 放置感測器、放置砲塔、建立連線並啟動短波次 |
| 感測器 | SENSOR | FAST / STEALTH | 前段偵測與快速敵人壓力 |
| 砲塔 | CANNON | BASIC / BOSS | 砲塔射程、訊號門檻與終端輸出 |
| AND 閘 | AND | ARMORED | 對裝甲敵人的傷害優勢 |
| OR 閘 | OR | SWARM | 整合多個感測訊號，穩定處理雜兵 |
| XOR 閘 | XOR | ELITE | 對精英敵人的高傷害價值 |
| NAND 閘 | NAND | ARMORED / SIEGE | 反制高護甲壓力 |
| 感知器 | PERCEPTRON | MIXED | 連續訊號與混合敵人情境 |
| 多線防守 | LANE CONTROL | MULTI-LANE | 觀察多條路線並分區補強 |

教學進行中會顯示目前章節、步驟目標、提示、進度與地圖高亮。左側元件按鈕會依目前步驟鎖定，禁止的元件會變暗並在點擊時顯示原因；波次按鈕與 `Space` 也只有在教學步驟允許時才會啟動短波次。

教學中的快捷鍵優先權和一般遊戲不同：

| 操作 | 教學中的功能 |
| --- | --- |
| `Esc` | 隨時開啟退出確認，可選擇繼續、回教學選單或回主選單 |
| `R` | 重開目前章節，不會重置成一般遊戲 |
| `P` | 暫停或繼續；暫停時仍可按 `Esc` 退出 |
| `H` | 開啟或關閉說明頁，關閉後回到教學 |
| `N` | 跳過可跳過的教學步驟 |
| `Space` | 只有在目前步驟允許時啟動教學短波次 |
| `T` | 依章節允許切換熱圖；未允許時顯示教學提示 |
| `A` | 依章節允許切換 AI 提示；未允許時顯示教學提示 |

教學使用固定場景與指定敵人，不會觸發一般隨機波次事件、TRAINING 獎勵或 GAMEOVER 流程，也不會寫入一般高分。

## 元件說明

| 元件 | 代號 | 費用 | 作用 |
| --- | --- | ---: | --- |
| 感測器 | `SEN` | 40 CR | 偵測範圍內敵人，輸出 0 到 1 的訊號 |
| 感知器 | `PCT` | 90 CR | 以加權求和與 Sigmoid 輸出連續訊號，可跨波學習 |
| AND 閘 | `AND` | 60 CR | 全部輸入大於 0.5 才輸出，對裝甲敵人有加成 |
| OR 閘 | `OR` | 60 CR | 任一輸入大於 0.5 即輸出，適合大量雜兵 |
| XOR 閘 | `XOR` | 60 CR | 恰好一個輸入大於 0.5 時輸出，對精英敵人有效 |
| NAND 閘 | `NND` | 70 CR | 非 AND 邏輯，後期可用來反制裝甲 |
| 砲塔 | `GUN` | 80 CR | 訊號大於 0.5 時自動攻擊範圍內敵人 |
| CPU | `CPU` | 0 CR | 核心目標，敵人抵達後會扣生命與 CPU HP |

連線限制：

- 砲塔是終端節點，不能輸出。
- CPU 不能被連入或連出。
- 感測器不能被連入。
- 連線不能形成迴圈。
- 單一元件最多可輸出 8 條連線。

## 敵人說明

| 敵人 | 特性 | 對應重點 |
| --- | --- | --- |
| 基礎兵 | 標準血量與速度，前期主要敵人 | 用感測器接砲塔建立穩定火力即可處理 |
| 快速兵 | 血量較低但移動快，Wave 4 起可能具備隱形能力 | 需要前段偵測與快速反應，避免太晚開火 |
| 裝甲兵 | 血量高、減傷高、速度慢 | AND / NAND 與破甲效果特別有效 |
| 精英兵 | 血量與威脅較高，Wave 10 起會帶護盾，並會定期再生 | XOR、標記與集中火力可降低突破率 |
| BOSS | 高血量、高傷害，具備狀態機行為，低血量時會進入 Rampage | 需要完整訊號鏈、破甲、標記與多座砲塔集火 |

敵人可能帶有特殊標籤：

- `BRUTE`：血量提高，移動速度下降，擊殺獎勵增加。
- `SWIFT`：移動速度提高，血量下降。
- `BOUNTY`：擊殺獎勵提高。

波次事件會改變敵人組成或規則：

- 蜂群入侵：大量低血量敵人，生成速度較快。
- 裝甲洪流：本波以裝甲兵為主。
- 霧戰 / 通訊中斷：敵人會隱形；通訊中斷會降低感測器有效範圍。
- 精英突擊：本波以精英兵為主，數量較少但壓力較高。
- 護衛 Boss：第一隻為 Boss，後續護衛會分散到 active lanes。
- 再生軍：非精英敵人也會週期性回血。
- 神速衝鋒：敵人速度大幅提高，多線後速度加成會略降以保留反應空間。
- 變異體入侵：血量與速度提高，並具備再生。
- 圍城艦來襲：第一隻敵人會變成高血量、高護甲、低速的超重單位。

敵方 AI 會在波次結束後學習兵種與路線存活率。Wave 10 起敵人會分散到多條進攻路線；Wave 16 起可同時四線進攻；Wave 30 起進入最多 6 條 active lanes 的高壓多線模式。每波前 N 個有效生成會先用 round-robin 覆蓋 N 條 active lanes，之後才依敵方 AI 的路線偏好加權分配。

## 基本玩法建議

1. 先放置感測器覆蓋路線，再把感測器訊號接到邏輯閘或感知器。
2. 將邏輯閘或感知器輸出接到砲塔，砲塔收到大於 0.5 的訊號才會開火。
3. 波次結束後觀察右側的敵情分析、AI 建議與威脅熱圖，補強高壓路段。
4. Wave 10 後會出現雙路徑，Wave 16 起進入四線同步進攻，避免所有火力只集中在單一路線。
5. 面對裝甲或圍城艦時，優先使用 AND / NAND 與破甲相關技能。

## 常見問題

### CMake 找不到 raylib

一般情況下不需要手動安裝 raylib，CMake 會自動下載。若下載失敗，請確認 Git 與網路連線可用，或先在系統中安裝 raylib。

### Linux 編譯時缺少 OpenGL / X11 相關函式庫

raylib 在 Linux 上通常需要 OpenGL、X11 與音訊相關開發套件。以 Debian / Ubuntu 為例，可先安裝常見依賴：

```bash
sudo apt update
sudo apt install build-essential cmake git libasound2-dev libx11-dev libxrandr-dev libxi-dev libgl1-mesa-dev libglu1-mesa-dev libxcursor-dev libxinerama-dev
```

### 中文顯示異常

專案會優先載入資料夾內的 `font.ttf`。若字型檔遺失，程式會嘗試載入系統中文字型；仍失敗時會退回 raylib 預設字型，中文可能無法完整顯示。
